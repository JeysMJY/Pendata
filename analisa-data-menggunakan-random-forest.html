Analisa Data Menggunakan Random Forest#

Pendahuluan#

1. Assembly Learning#

Assembly Learning adalah sebuah teknik atau “seni” dalam machine learning yang menggabungkan sekumpulan model dasar (base learners) yang beragam untuk membangun satu model akhir dengan stabilitas dan akurasi yang jauh lebih tinggi dibandingkan model individual (seperti satu Decision Tree).

Untuk menghasilkan kumpulan base learner yang beragam, setiap model dalam assembly dapat dibangun menggunakan:

  • Algoritma yang berbeda.

  • Hyperparameter yang berbeda.

  • Representasi fitur yang berbeda.

  • Set data pelatihan yang berbeda.

Alur Kerja Assembly Learning

Proses pembagian dan penggabungan data dalam Assembly dapat divisualisasikan sebagai berikut:

  • D (Data Asli): Dataset utuh yang kita miliki.

  • D1, D2, …, Dn (Pemecahan Data): Subset data yang dipecah dari data asli. Jika semua subset digabungkan, ia merepresentasikan data asli.

  • C1, C2, …, Cn (Pembangunan Model): Melatih model pengklasifikasi pada masing-masing subset data.

  • C (Combine Classifiers): Menggabungkan hasil dari semua model (C1…Cn) untuk memilih keputusan terbaik (Majority Voting).

2. Memahami Model Error (Bias & Varians)#

Kesalahan (error) yang muncul dari model machine learning manapun dapat dipecah menjadi tiga komponen matematis:

\[ Err(x) = Bias^2 + Variance + Irreducible\ Error \]

Dalam mengevaluasi model (misalnya menggunakan Mean Absolute Error atau MAE, di mana kita menghitung selisih absolut aktual dan prediksi \(|x|\)), kita berhadapan dengan Trade-off antara Bias dan Varians:

  • Bias (Kesalahan Bias): Mengukur seberapa besar rata-rata nilai prediksi meleset dari nilai aktual.

    • High Bias (Underfitting): Model berkinerja buruk karena gagal menangkap tren atau pola dari data saat ini. Model terlalu sederhana.

    • Low Bias: Model sangat akurat dalam mempelajari pola data latih (error antara prediksi dan aktual sangat rendah).

  • Varians (Sebaran Data): Mengukur seberapa besar perbedaan prediksi yang dibuat untuk observasi yang sama jika data pelatihannya sedikit diubah.

    • High Variance (Overfitting): Prediksi sangat tersebar (berjauhan). Model terlalu “menghafal” data latih sehingga gagal memprediksi observasi baru di luar data latih tersebut.

    • Low Variance: Selisih prediksi satu sama lain tidak berjauhan dan stabil.

Tujuan Utama: Mengharapkan model yang memiliki Low Bias (prediksi akurat/dekat dengan aktual) dan Low Variance (prediksi konsisten dan stabil pada data baru).

3. Bootstrap & Bagging (Bootstrap Aggregating)#

Untuk mengatasi masalah High Variance pada Decision Tree tanpa merusak Low Bias yang sudah dimilikinya, kita menggunakan metode Bagging.

Bootstrap: Teknik pengambilan data dari dataset asli secara acak dengan pengembalian (with replacement).

Bagging: Menggunakan dataset dari hasil Bootstrap tersebut untuk melatih banyak model secara paralel.

  • Dengan merata-ratakan atau memvoting hasil dari banyak model yang memiliki Low Bias, Bagging secara efektif meredam High Variance. Sistem ini mengandalkan parameter estimator (banyaknya set data) dan n_estimators (banyaknya pohon).

4. Random Forest#

Random Forest adalah algoritma assembly yang secara spesifik dibangun dengan menggabungkan banyak model Decision Tree.

Algoritma ini adalah wujud nyata dari pencapaian model yang ideal. Karena setiap pohon keputusan di dalamnya dibiarkan tumbuh dalam (yang memastikan Low Bias), dan digabungkan menggunakan teknik Bagging serta keacakan fitur (Feature Randomness), Random Forest secara drastis menurunkan varians.

Hasil akhirnya? Random Forest sukses mencapai “Holy Grail” dalam machine learning: sebuah model tangguh yang memiliki Low Bias (akurat) sekaligus Low Variance (stabil dan tidak overfitting pada data baru).

Data Preparation#

Pengertian#

Fase ini sering disebut “data mining”, yaitu menyiapkan kumpulan data akhir untuk pemodelan. Kegiatan pada fase ini adalah memperbaiki kualitas data agar sesuai dengan proses modeling yang akan dilakukan berikutnya.

Data preparation penting karena kualitas data sangat memengaruhi hasil model Decision Tree. Jika data masih memiliki nilai anomali, missing value, atau tipe data target yang tidak sesuai, maka proses modeling dapat menghasilkan model yang kurang baik.

2. Mengubah Tipe Data Target#

Setelah itu, saya merubah tipe data kolom “Outcome” yang merupakan Label atau Target. Pada KNIME jika label merupakan integer maka tidak bisa dilakukan pengecekan akurasinya, sehingga saya merubahnya menjadi string tetapi tetap dengan 2 kelas yaitu “0” dan “1”.

Pada KNIME untuk implementasikan hal tersebut menggunakan node “Number to String” untuk merubah tipe data integer ke string. Memindahkan semua kolom selain “Outcome” ke Excludes agar kolom lainnya tidak dirubah menjadi String.

Kolom Outcome diubah dari tipe numerik menjadi string. Tujuannya agar KNIME membaca Outcome sebagai class target, bukan sebagai angka biasa.

Modeling Random Forest#

Pengertian Modeling#

Modeling adalah tahap membangun model machine learning menggunakan data yang sudah melalui proses preparation. Pada tahap ini, data training digunakan untuk melatih model agar dapat mempelajari pola dari fitur-fitur yang tersedia.

Pada project ini, menggunakan script python dengan memanfaatkan library scikit-learn yaitu from sklearn.ensemble import RandomForestClassifier

Implementasi script python#

Berikut adalah skrip untuk melatih dan membandingkan bagaimana model assembly (Random Forest) mempertahankan akurasi sekaligus menstabilkan prediksi dibandingkan Decision Tree tunggal.

Parameter yang digunakan:

  • n_estimators=50: Jumlah pohon yang dibuat. (Banyak pohon = prediksi makin stabil).

  • random_state=0: Mengunci keacakan agar hasil akurasi model selalu sama setiap kali dijalankan ulang.

  • criterion=”gini”: Rumus untuk mencari pemisahan data terbaik pada setiap cabang agar hasilnya paling murni (satu kelas).

  • max_depth=10: Batas maksimal kedalaman pohon (10 tingkat). Fungsinya untuk mencegah overfitting (model terlalu menghafal).

  • min_samples_leaf=2: Syarat minimal harus ada 2 data di ujung cabang terakhir. Fungsinya mencegah pohon membuat aturan rumit hanya untuk 1 data (mencegah overfitting).

import pandas as pd
import knime.scripting.io as knio
from sklearn.ensemble import RandomForestClassifier
import pickle

# Input 0 (Panah Atas) = Data Training
df_train = knio.input_tables[0].to_pandas()

# Input 1 (Panah Bawah) = Data Testing
df_test = knio.input_tables[1].to_pandas()

# Mengambil kolom fitur dan target dari data Training
X_train = df_train.drop(columns=['Outcome'])
y_train = df_train['Outcome']

# Mengambil kolom fitur dan target dari data Testing
X_test = df_test.drop(columns=['Outcome'])
y_test = df_test['Outcome']

# Inisialisasi Random Forest Classifier dengan 50 decision trees
rf_model = RandomForestClassifier(n_estimators= 50, random_state=0, criterion="gini", max_depth=10, min_samples_leaf=2)

# Model belajar hanya menggunakan data dari port atas (X_train & y_train)
rf_model.fit(X_train, y_train)

# Model menguji kemampuannya menebak data dari port bawah (X_test)
y_pred = rf_model.predict(X_test)

# Membuat DataFrame berisi kolom nilai asli vs nilai hasil tebakan model

scorer_df = pd.DataFrame({
    'Actual_Outcome': y_test.values,   # Data asli dari dataset test
    'Predicted_Outcome': y_pred        # Hasil prediksi model Random Forest
})

# Mengubah model menjadi format byte binary sesuai dengan modul latihan
model_bytes = pickle.dumps(rf_model)

knio.output_tables[0] = knio.Table.from_pandas(scorer_df)

# Kirim objek model ke Port Objek 0
knio.output_objects[0] = model_bytes

Kesimpulan#

Berdasarkan hasil evaluasi model menggunakan confusion matrix dan beberapa metrik performa, model Random Forest menunjukkan hasil yang lebih baik dibandingkan Decision Tree.

Hal ini dapat dilihat dari nilai:

  • Accuracy Random Forest lebih tinggi yaitu 76.6%, sedangkan Decision Tree hanya 68.8%.

  • Recall/Sensitivity Random Forest lebih baik dalam mendeteksi kasus positif diabetes.

  • Specificity Random Forest lebih tinggi, sehingga lebih baik dalam mengenali data negatif dengan benar.

  • F1-Score Random Forest juga lebih tinggi, yang menunjukkan keseimbangan performa klasifikasi yang lebih baik.

  • Nilai Cohen’s Kappa Random Forest lebih besar, menandakan tingkat kesepakatan prediksi model yang lebih baik.

Dari confusion matrix juga terlihat bahwa Random Forest menghasilkan:

  • jumlah prediksi benar lebih banyak,

  • false negative lebih sedikit,

  • dan false positive lebih rendah dibandingkan Decision Tree.

Dengan demikian, dapat disimpulkan bahwa Random Forest merupakan model yang lebih optimal dan memiliki performa klasifikasi yang lebih baik untuk dataset diabetes yang digunakan pada penelitian ini.
